Snake case to camel case and back using regular expressions and Python

Snake case and Camel case are conventions of naming variables, functions and classes. Most teams and projects prescribe a particular case in their style guides. Examples of camel case: MyClass MyClassFactory MyClassFactoryBuilder MyClassFactoryBuilderImpl myInstance myInstance2 abc patternMatcher Examples of snake case: add matrix_add diagonal_matrix_add pseudo_inverse If we want to convert back and forth between these cases, we must look for the points of interest - the word boundaries. Camel case boundaries have the first letter capitalized while the snake case word boundaries have an _. Snake case to camel case Here is a regular expression for finding out the _ and the first letter in the next word: (.*?)_([a-zA-Z]) This regex has 2 parts: (.*?) finds everything upto the _. The ‘.’ means any character. ‘*’ stands for match 0 or more instances ‘?’ stands for non-greedy match.
